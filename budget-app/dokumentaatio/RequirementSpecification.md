# Requirements Specification



## Application purpose



The application allows the users to track their monthly spendings and incomes. The application can be used by many users.



## Users



The application allows to create an account. Users can log their incomes and spendings and later review them. 



## Application UI



The application has a log in page, a create an account and the page to view your own transactions. The transactions can be viewed only when logged in.



## Basic system



#### Before logging in



* A user can create an account.
* The username has to be unique
* A user can log in only if the username exists and the password is correct.
* If does not exist, the application shows an error message.



#### After logging in



* User can add a transaction
* A user can view their prior transactions
* A user can delete or edit their transactions
* A user can log out
* A created transaction can be viewed only by the creator
