# Requirements Specification



## Application purpose



The application allows the users to track their spendings and incomes. The application can be used by many users.



## Users



* The application allows to create an account. DONE
* Users can log their incomes and spendings. DONE
* Users can review their transactions. DONE
* Users can delete their logged information. DONE 
* User can see how much was spent and how much was made all together. DONE



## Application UI



The application has a log in page, a create an account, the page to view your own transactions and a page to create a new transaction. The transactions can be viewed only when logged in, and the user can see only their own transactions and a general summary of how much was spent all together and how much was saved.



## Basic system



#### Before logging in



A user can create an account, but the username has to be unique, and have to make a password. A user can log in only if the username exists and the password is correct. If the account does not exist, the application shows an error message, and will ask if they would like to make an account.



#### After logging in



User can add a transaction, such as "payed rent" or "this months salary". A user can view their prior transactions, and a summary of them. A user can delete or edit their transactions. A user can log out. And most importantly created transaction can be viewed only by the creator, when they are logged in.
